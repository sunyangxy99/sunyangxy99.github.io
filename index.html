//加密函数
public static File encryption(String path,String password){
File infile=new File(path);
File outfile=new File(infile.getName());
BufferedInputStream in=null;
BufferedOutputStream out=null;
try{
in=new BufferedInputStream(new FileInputStream(infile));
out=new BufferedOutputStream(new FileOutputStream(outfile));
byte[] pawbytes=password.getBytes();
int len=0;
int i=0;
while((len=in.read())!=-1){
byte b=(byte)len;
out.write(b^pawbytes[i]);
i++;
if(i==pawbytes.length)
i=0;}
return outfile;
}catch(IOException e){
e.printStackTrace();}
catch(NullPointerException e){
e.printStackTrace();
}finally{
try{if(in!=null)
in.close();
if(out!=null)
out.flush();
out.close();}
catch(IOException e){
e.printStackTrace();
}
}
return null;
}
//解密函数
public static File deciphering(String path,String password){
return encryption(path,password);
}
}

HTTPS

发表于 2017-07-29

　　https:是一种以安全为目的的http通道，也即安全版本的http协议。是在http协议下加入了ssl/tls协议层。
其作用是①对传输的内容进行加密，②对站点可以进行身份认证（确认网站的真实性），③还可以保证数据的完整性，使得一些抓包工具即使抓取到网站的信息也无法进行欺诈的行为。

　　其缺点是①第一次登陆网站比http慢，是因为握手阶段比较费时；②https连接缓存不如http高效，会增加数据开销和功耗；③ssl证书需要钱，ssl证书通常需要绑定ip，不能在同一ip上绑定多个域名，ipv4资源不可能支撑这个消耗。④https协议的加密范围也比较有限，在黑客攻击、拒绝服务攻击、服务器劫持等方面几乎起不到什么作用。最关键的，ssl证书的信用链体系并不安全，特别是在某些国家可以控制CA根证书的情况下，中间人攻击一样可行。
　　
　　http与https的区别：①https需要到CA申请证书；②http是明文传输，https是具有安全性的ssl加密传输协议；③http的端口是80，https默认端口是443；④http是一种无状态的连接，https协议是由http+ssl协议构建的可进行加密传输、身份认证的网络协议，比http协议安全。
